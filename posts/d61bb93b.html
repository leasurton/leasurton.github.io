ICS DataLab 题目及解析

发表于2023-10-31|更新于2023-10-31
|阅读量:

DataLab题目列表、解析以及代码

bitXor

bitXor - x^y using only ~ and &
Example: bitXor(4, 5) = 1
Legal ops: ~ &
Max ops: 14
Rating: 1

用 ~ 和 & 表示 | ，再用~ & | 表示 ^
摩尔根定理；~(~x & ~y)= x | y
比较x&y 与 ~x&~y的结果

thirdBits

thirdBits - return word with every third bit (starting from the LSB) set to 1
Legal ops: ! ~ & ^ | + << >>
Max ops: 8
Rating: 1

每三位置1，则预先设置0100 1001进行移位，左移时右边空缺补0，空缺位置与未移位前进行|运算则可以得到补足。
由于0100 1001是满足要求的其中一个字节，同时左移时右边空缺补0，则可以得知移位后仅需使得相邻两个字节之间，仍然满足每三位置1
则进行n次移位操作后，可以得到满足题目要求的结果

fitsShort

fitsShort - return 1 if x can be represented as a
16-bit, two’s complement integer.
Examples: fitsShort(33000) = 0, fitsShort(-32768) = 1
Legal ops: ! ~ & ^ | + << >>
Max ops: 8
Rating: 1

是否能表示16位二进制补码整数，要看是否在范围内-32768~32767
正数最大为 0111 1111 1111 1111 负数最大为1000 0000 0000 0000
正数负数的符号位不同即后16位的最高位不同
但是int类型不止16位，则对于int x前置有其符号位
根据这个关系进行判断
如果x不在-32768~32767范围内但除后16位外全是符号位，此时则看其在16位的最> 高位与其在int类型的符号位是否相同
如果x不在-32768~32767范围内且除后16位外并非全是符号位，此时 x>>15 与 x>>16的 ^ 则会出现不是在最后一位的1，此时取!则为0

isTmax

isTmax - returns 1 if x is the maximum, two’s complement number,
and 0 otherwise
Legal ops: ! ~ & ^ | +
Max ops: 10
Rating: 1

max 0111 1111 1111 1111 没有更高位
区别max 与 例如 0011 1111 1111 1111 则需要判断最高位，通过x+1，若最高位变化，则max
若x加上1后与x本身进行 ^ 运算，则没有变化的位置取1，其余取0，max为全1
而需要排除全1的影响，则全1按位取反后为0，其余情况不为0，但无法使用逻辑或||，所以只能逻辑取反，全1为1，其余为0
（同时由于输出为0或1，只能通过逻辑取反做到）
如此与所需要的返回值相异，将排除全1的判断与区别max与其余情况的判断整合后需再次逻辑取反
因此区别max与其余情况的判断也需要与所需要的返回值相异，才能在逻辑取反后符合要求
所以需要使max在区别操作中为全0，x+1按位取反再与x进行 ^ 操作

max 0111 1111 1111 1111 没有更高位,加1(或者取反)则变为min 1000 0000 0000 0000（这里不知道为什么加1不通过，~x也不通过）×，不能用移位运算
所以用x+1与1<<31比较，异或，如果所有位都相同，则x为max，得到值0，否则不为max，得到非0值
再通过逻辑取反!返回0或1

fitsBits

fitsBits - return 1 if x can be represented as an
n-bit, two’s complement integer.
1 <= n <= 32
Examples: fitsBits(5,3) = 0, fitsBits(-4,3) = 1
Legal ops: ! ~ & ^ | + << >>
Max ops: 15
Rating: 2

upperBits

upperBits - pads n upper bits with 1’s
You may assume 0 <= n <= 32
Example: upperBits(4) = 0xF0000000
Legal ops: ! ~ & ^ | + << >>
Max ops: 10
Rating: 1

anyOddBit

anyOddBit - return 1 if any odd-numbered bit in word set to 1
Examples anyOddBit(0x5) = 0, anyOddBit(0x7) = 1
Legal ops: ! ~ & ^ | + << >>
Max ops: 12
Rating: 2

byteSwap

byteSwap - swaps the nth byte and the mth byte
Examples: byteSwap(0x12345678, 1, 3) = 0x56341278
byteSwap(0xDEADBEEF, 0, 2) = 0xDEEFBEAD
You may assume that 0 <= n <= 3, 0 <= m <= 3
Legal ops: ! ~ & ^ | + << >>
Max ops: 25
Rating: 2

absVal

absVal - absolute value of x
Example: absVal(-1) = 1.
You may assume -TMax <= x <= TMax
Legal ops: ! ~ & ^ | + << >>
Max ops: 10
Rating: 4

divpwr2

divpwr2 - Compute x/(2^n), for 0 <= n <= 30
Round toward zero
Examples: divpwr2(15,1) = 7, divpwr2(-33,4) = -2
Legal ops: ! ~ & ^ | + << >>
Max ops: 15
Rating: 2

float_neg

float_neg - Return bit-level equivalent of expression -f for
floating point argument f.
Both the argument and result are passed as unsigned int’s, but
they are to be interpreted as the bit-level representations of
single-precision floating point values.
When argument is NaN, return argument.
Legal ops: Any integer/unsigned operations incl. ||, &&. also if, while
Max ops: 10
Rating: 2

logicalNeg

logicalNeg - implement the ! operator, using all of
the legal operators except !
Examples: logicalNeg(3) = 0, logicalNeg(0) = 1
Legal ops: ~ & ^ | + << >>
Max ops: 12
Rating: 4

bitMask

bitMask - Generate a mask consisting of all 1’s
lowbit and highbit
Examples: bitMask(5,3) = 0x38
Assume 0 <= lowbit <= 31, and 0 <= highbit <= 31
If lowbit > highbit, then mask should be all 0’s
Legal ops: ! ~ & ^ | + << >>
Max ops: 16
Rating: 3

isGreater

isGreater - if x > y then return 1, else return 0
Example: isGreater(4,5) = 0, isGreater(5,4) = 1
Legal ops: ! ~ & ^ | + << >>
Max ops: 24
Rating: 3

logicalShift

logicalShift - shift x to the right by n, using a logical shift
Can assume that 0 <= n <= 31
Examples: logicalShift(0x87654321,4) = 0x08765432
Legal ops: ! ~ & ^ | + << >>
Max ops: 20
Rating: 3

satMul2

satMul2 - multiplies by 2, saturating to Tmin or Tmax if overflow
Examples: satMul2(0x30000000) = 0x60000000
satMul2(0x40000000) = 0x7FFFFFFF (saturate to TMax)
satMul2(0x60000000) = 0x7FFFFFFF (saturate to TMax)
Legal ops: ! ~ & ^ | + << >>
Max ops: 20
Rating: 3

subOK

subOK - Determine if can compute x-y without overflow
Example: subOK(0x80000000,0x80000000) = 1,
subOK(0x80000000,0x70000000) = 0,
Legal ops: ! ~ & ^ | + << >>
Max ops: 20
Rating: 3

trueThreeFourths

trueThreeFourths - multiplies by 3/4 rounding toward 0,
avoiding errors due to overflow
Examples: trueThreeFourths(11) = 8
trueThreeFourths(-9) = -6
trueThreeFourths(1073741824) = 805306368 (no overflow)
Legal ops: ! ~ & ^ | + << >>
Max ops: 20
Rating: 4

isPower2

isPower2 - returns 1 if x is a power of 2, and 0 otherwise
Examples: isPower2(5) = 0, isPower2(8) = 1, isPower2(0) = 0
Note that no negative number is a power of 2.
Legal ops: ! ~ & ^ | + << >>
Max ops: 20
Rating: 4

float_i2f

float_i2f - Return bit-level equivalent of expression (float) x
Result is returned as unsigned int, but
it is to be interpreted as the bit-level representation of a
single-precision floating point values.
Legal ops: Any integer/unsigned operations incl. ||, &&. also if, while
Max ops: 30
Rating: 4

howManyBits

howManyBits - return the minimum number of bits required to represent x in
two’s complement
Examples: howManyBits(12) = 5
howManyBits(298) = 10
howManyBits(-5) = 4
howManyBits(0) = 1
howManyBits(-1) = 1
howManyBits(0x80000000) = 32
Legal ops: ! ~ & ^ | + << >>
Max ops: 90
Rating: 4

float_half

float_half - Return bit-level equivalent of expression 0.5*f for
floating point argument f.
Both the argument and result are passed as unsigned int’s, but
they are to be interpreted as the bit-level representation of
single-precision floating point values.
When argument is NaN, return argument
Legal ops: Any integer/unsigned operations incl. ||, &&. also if, while
Max ops: 30
Rating: 4
